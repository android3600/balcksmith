# 대장장이 키우기 프로토타입 개선 정리

## 작업 개요

Claude가 만든 기존 단일 HTML 프로토타입을 기준으로, 게임의 핵심 로직은 유지하면서 플레이 중 판단이 더 잘 보이도록 UI와 흐름을 개선했다.

이번 작업의 목표는 대규모 기능 추가가 아니라, 사용자가 바로 체감할 수 있는 완성도 향상이다.

## 개선 방향

- 기존 만족도, 점수, 생명, 리롤, 콤보, 난이도 공식은 유지
- 손님 의뢰 조건을 더 명확하게 표시
- 슬롯 결과가 좋은지 나쁜지 즉시 판단할 수 있게 표시
- 리롤을 언제 써야 하는지 플레이어가 이해하기 쉽게 개선
- 모바일 세로 화면에서 정보가 한눈에 들어오도록 정리

## 주요 변경 사항

### 1. 의뢰 패널 추가

손님 정보 아래에 의뢰 패널을 추가했다.

표시 정보:

- 손님 등급
- 무기
- 직업
- 목표 만족도
- 현재 만족도
- 목표까지 남은 만족도
- 남은 리롤 횟수

이를 통해 플레이어가 현재 라운드의 목표를 계속 확인할 수 있다.

### 2. 슬롯 판단 패널 추가

슬롯이 공개된 뒤, 현재 슬롯을 확정할지 리롤할지 판단할 수 있도록 별도 패널을 추가했다.

판정 종류:

- 대박: 높은 티어 슬롯
- 적중: 무기와 직업에 잘 맞는 슬롯
- 무난: 어느 정도 기여하는 슬롯
- 빗나감: 만족도에 기여하지 않는 슬롯
- 저주 / 저주+: 손님 성향에 따라 위험하거나 강한 슬롯

### 3. 슬롯 카드 라벨 개선

확정된 슬롯 카드 안에도 판정 라벨을 표시하도록 수정했다.

이전에는 티어와 속성만 보여서 결과 해석이 어려웠지만, 이제 슬롯마다 `적중`, `무난`, `빗나감` 같은 결과가 함께 표시된다.

### 4. 시작 화면 문구 정리

시작 화면 설명을 짧고 명확하게 정리했다.

기존에는 규칙 설명이 다소 길었고 프로토타입 느낌이 강했으나, 현재는 핵심 플레이 방식과 리롤 규칙만 빠르게 전달한다.

## 검증 결과

### 시뮬레이터 검증

명령:

```bash
node simulate.mjs
```

결과:

- 평균 만족도: 43.7
- 초급 클리어율: 96.3%
- 중급 클리어율: 81.6%
- 고급 클리어율: 29.8%
- 특급 클리어율: 7.4%

기존 SPEC 기준 범위와 크게 벗어나지 않아, UI 개선으로 인해 밸런스 공식이 깨지지 않았음을 확인했다.

### 브라우저 검증

모바일 폭 기준으로 아래 흐름을 확인했다.

- 게임 시작
- 첫 슬롯 제련
- 슬롯 판정 패널 표시
- 리롤 1회 사용
- 리롤 횟수 0회로 갱신
- 다음 슬롯 진행

콘솔에는 앱 오류가 없었고, `favicon.ico` 404만 확인됐다.

## 현재 상태

프로토타입의 핵심 루프는 유지하면서, 플레이어가 현재 상황을 이해하고 리롤을 판단하는 경험이 개선됐다.

다음 개선 후보는 아래와 같다.

- 최종 결과 화면 polish
- 슬롯 연출 속도 조절
- 재료/골드/업그레이드 같은 v2 성장 요소 추가
- 모바일 작은 화면에서 더 촘촘한 레이아웃 최적화
